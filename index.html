Introdução

Este website possui a finalidade de apresentar alguns padrões de projeto de interface, suas descrições, seus casos de uso e exemplos, ao mesmo tempo que faz o uso de recomendações de acessibilidade presentes no e-MAG (Modelo de Acessibilidade em Governo Eletrônico) Versão 3.1.

Foi desenvolvido como requisito parcial à aprovação na disciplina de Design de Interface Humano Computador, ministrada pela Profa. Dra. Andreia Marini no Instituto Federal do Paraná Campus Palmas.

Padrões de Projeto de Interface

São estruturas de interação que foram analisadas, estudadas e aceitas em determinados ambientes, devido ao seu grau de repetição (prática comum) e verificação de funcionamento.

“Bons padrões de interface humano computador são os que favorecem tantos aos desenvolvedores, proporcionando-lhes eficiência na programação, como aos usuários, oferecendo-lhes usabilidade nas interações com o sistema.” (CYBIS, BETIOL, FAUST, 2015)

e-MAG

O governo brasileiro, comprometido com a inclusão, buscou, através da elaboração do Modelo de acessibilidade do governo eletrônico, facilitar o acesso para todas as pessoas às informações e serviços disponibilizados nos sítios e portais do governo.

O eMAG foi desenvolvido em 2004 baseado no estudo de 14 normas existentes em outros países acerca de acessibilidade digital. Dentre as normas analisadas estavam a Section 508 do governo dos Estados Unidos, os padrões CLF do Canadá, as diretrizes irlandesas de acessibilidade e documentos de outros países como Portugal e Espanha.

Recomendações de Acessibilidade

Recomendação 1.1

Respeitar os padrões web: fazendo o uso de boas práticas, usando HTML5, CSS, SVG, AJAX, e realizando separação de camadas. Assim, para a camada de conteúdo devem ser utilizadas as linguagens de marcação, como HTML e xHTML. Para a camada de apresentação visual do conteúdo, utilizam-se as folhas de estilo css em qualquer uma de suas versões. Já para a camada que modifica o comportamento dos elementos, são utilizadas linguagens javascript e modelos de objeto (dom).

Recomendação 1.2

Organizar o código HTML de forma lógica e semântica: ou seja, apresentando os elementos em uma ordem compreensível e correspondendo ao conteúdo desejado. Cada elemento HTML deve ser utilizado para o fim que ele foi criado. Assim, marcação semântica adequada deve ser utilizada para designar os cabeçalhos (h1, h2, h3), as listas (ul, ol, dl), texto enfatizado (strong), marcação de código (code), marcação de abreviaturas (abbr), marcação de citações longas (blockquote), etc

Recomendação 1.3

Utilizar corretamente os níveis de cabeçalho: os níveis de cabeçalho (elementos HTML H1 a H6) devem ser utilizados de forma hierárquica, pois organizam a ordem de importância e subordinação dos conteúdos, facilitando a leitura e compreensão.

Recomendação 1.5

Fornecer âncoras para ir direto a um bloco de conteúdo: devem ser fornecidas âncoras, disponíveis na barra de acessibilidade, que apontem para links relevantes presentes na mesma página. Assim, é possível ir ao bloco de conteúdo desejado.

Recomendação 1.7

Separar links adjacentes: links adjacentes devem ser separados por mais do que simples espaços, para que não fiquem confusos, em especial para usuários que utilizam leitor de tela. Para isso, é recomendado o uso de listas, onde cada elemento dentro da lista é um link. As listas podem ser estilizadas visualmente com CSS para que os itens sejam mostrados da maneira desejada, como um ao lado do outro.

Recomendação 3.1

Identificar o idioma principal da página: deve-se identificar o principal idioma utilizado nos documentos. A identificação é feita por meio do atributo lang do HTML e, para documentos XHTML, é utilizado o xml:lang. Ele deve ser declarado em todas as páginas, pois além de auxiliar na acessibilidade do conteúdo, também permite melhor indexação pelos motores de busca.

Recomendação 3.3

Oferecer um título descritivo e informativo à página: o título da página deve ser descritivo e informativo, devendo representar o conteúdo principal da página, já que essa informação será a primeira lida pelo leitor de tela, quando o usuário acessar a página.

Recomendação 3.6

Fornecer alternativa em texto para as imagens do sítio: deve ser fornecida uma descrição para as imagens da página, utilizando-se, para tanto o atributo alt. No caso de banners e outras imagens que reproduzam textos, o ideal é reproduzir o texto escrito.

Recomendação 4.1

Oferecer contraste mínimo entre plano de fundo e primeiro plano as cores do plano de fundo e do primeiro plano deverão ser suficientemente contrastantes para que possam ser visualizadas, também, por pessoas com baixa visão, com cromodeficiências ou que utilizam monitores de vídeo monocromático. Não deverão ser utilizadas imagens atrás do texto (background), pois acabam por dificultar a leitura e desviar a atenção do usuário.

Recomendação 4.3

Permitir redimensionamento sem perda de funcionalidade: a página deve continuar legível e funcional mesmo quando redimensionada para até 200%. Assim, é preciso garantir que, quando a página for redimensionada, não ocorram sobreposições nem o aparecimento de uma barra horizontal.

Padrões de Projeto de Interface

Carousel

Descrição:Uma lista de itens é organizada de maneira horizontal, por meio da qual o usuário pode visualizar e navegar pelos itens de maneira fácil e rápida

Resumo do Problema:"O usuário precisa navegar por uma lista de itens e possivelmente selecionar um deles"

Recomendações de Uso:

  • Use quando você tem uma grande quantidade de itens pra mostrar, mas quer permitir que o usuário se concentre em apenas alguns itens por vez;
  • Use quando você quer mostrar ao usuário que existem mais itens disponíveis do que apenas os que estão sendo mostrados atualmente;
  • Use quando você não tem espaço disponível para mostrar todos os itens de uma vez;
  • Use quando você tem itens essencialmente visuais a serem mostrados, como pôsteres de filmes, capas de álbuns, produtos, etc.;
  • Não use quando os itens não são visuais, como links para artigos, documentos de PDF, etc.
  • Não use quando o conteúdo ao qual está vinculado não pode ser imediatamente idenficado por uma imagem

    Pull to Refresh

    Descrição:O usuário faz um movimento de arrastar com o dedo, de cima para baixo, para atualizar a tela atual;

    Resumo do Problema:"O usuário quer recuperar mais dados ou atualizar conteúdos já disponíveis na tela";

    Recomendações de Uso:

    • Usado para atualizar qualquer conteúdo, como listas, listas de grade e coleções de cartões que são classificadas em ordem cronológica reversa;
      • Feed de Conteúdo (Facebook, Instagram, Twitter);
      • Inbox (Email, Mensagens de Texto);
    • Usado para atualizar uma página inteira ou um widget;
      • Página do Navegador;
    • Usado quando feito o design para interfaces touchscreen;

    Cards

    Descrição: Os cards são utilizados para agrupar informações e componentes de design em um container de tamanho flexível, que lembra visualmente uma carta de baralho, dando uma sensação de relevo, profundidade;

    Resumo do Problema: “O usuário precisa navegar por conteúdos de variados tipos e tamanhos”;

    Recomendações de Uso:

    • Usado para apresentar conteúdo composto de diferentes elementos;
    • Usado para mostrar elementos cujo tamanho ou ações variam, como por exemplo, fotos com legendas de tamanhos variados;
    • Usado para apresentar conteúdo que:
      • Em uma coleção, consiste de múltiplos tipos de dados (imagens, vídeos, texto);
      • Contém conteúdo interativo;
      • Suporta conteúdo de tamanho variável (legendas, comentários);
    • Usado para unir vários pedaços de informação sobre um único assunto para formar um pedaço de conteúdo coerente;
    • Recomendado quando usuários pesquisam por informações;
    • Funcionam melhor em coleções de itens heterogêneos (quando o conteúdo não é todo do mesmo tipo básico);
